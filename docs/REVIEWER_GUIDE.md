# 검토자 가이드

## 1. 검토의 의미

검토(Verification)란 코치가 제출한 **증빙서류의 진위성과 적합성을 확인**하는 과정입니다.

검토자는 코치가 입력한 역량 정보(학력, 자격증, 경력 등)에 대해 첨부된 증빙서류를 검토하여, 해당 정보가 사실임을 확인합니다.

### 검토의 목적
- 코치 역량 정보의 **신뢰성 확보**
- 프로젝트 선발 과정의 **공정성 보장**
- 허위 정보 입력 **방지**

### 검토 완료 시 효과
- **전역 검증 완료**: 검토 완료된 역량은 "확인완료" 상태가 되어, 코치가 다른 프로젝트에 지원할 때 재검토 없이 **자동으로 재사용**됩니다.
- 코치의 "전자지갑"에 검증된 역량으로 저장됩니다.

## 2. 검토 정책

### 2.1 AND 조건 (전체 컨펌 필수)

검토는 **OR 조건이 아닌 AND 조건**으로 운영됩니다.

| 정책 | 설명 |
|------|------|
| **필요 인원** | 시스템에서 설정된 검토자 수(기본: 2명)가 **모두** 컨펌해야 함 |
| **AND 조건** | 검토자 A만 컨펌하면 미완료, A와 B **모두** 컨펌해야 검토 완료 |
| **독립적 판단** | 각 검토자는 독립적으로 판단하여 컨펌 |

**예시**:
- 필요 검토자 수: 2명
- 검토자 A 컨펌: ✓ → 상태: 검토중 (1/2)
- 검토자 B 컨펌: ✓ → 상태: **검토완료** (2/2)

### 2.2 권한 및 역할

| 역할 | 권한 |
|------|------|
| **VERIFIER** | 검토 메뉴 접근, 컨펌/리셋 가능 |
| **SUPER_ADMIN** | 모든 검토 권한 + 검토자 수 설정 가능 |
| **PROJECT_MANAGER** | 해당 프로젝트 검토 가능 |

## 3. 검토 메뉴 프로세스 (사용법)

### 3.1 검토 화면 접근

1. 상단 메뉴에서 **"검토"** 클릭
2. 검토 대기 목록이 표시됨

### 3.2 검토자 인원수 설정

**SUPER_ADMIN 전용** 기능입니다.

1. **관리자 메뉴** → **시스템 설정**
2. `required_verifier_count` 값 수정
3. 기본값: **2명**

> **주의**: 검토자 수를 변경하면, 이후 제출되는 모든 증빙에 적용됩니다.

### 3.3 컨펌 (Confirm)

증빙서류가 적합하다고 판단될 때 사용합니다.

**프로세스**:
1. 검토 대기 목록에서 항목 선택
2. 증빙서류 확인 (상세보기/다운로드)
3. 검토사항 확인 (아래 4장 참조)
4. **"컨펌"** 버튼 클릭

**컨펌의 효과**:
- 해당 검토자의 컨펌 기록이 저장됨
- 필요 인원이 모두 컨펌하면 **자동으로 검토 완료** 처리
- 검토 완료 시 코치의 역량이 **전역 검증 완료** 상태로 변경

### 3.4 리셋 (Reset)

이미 컨펌된 항목을 초기화할 때 사용합니다.

**사용 상황**:
- 컨펌 후 오류 발견 시
- 증빙서류 재검토 필요 시
- 코치가 증빙서류를 교체한 경우

**프로세스**:
1. 검토 완료된 항목 선택
2. **"리셋"** 버튼 클릭
3. 리셋 사유 입력 (필수)
4. 확인

**리셋의 효과**:
- **모든 검토자**의 컨펌 기록이 삭제됨 (자신의 것만 삭제 아님)
- 해당 항목이 **검토 대기** 상태로 복귀
- 전역 검증 완료 상태가 **해제**됨
- 리셋 사유가 활동 기록에 남음

### 3.5 보완요청 (Supplement Request)

증빙서류가 부적합하여 코치에게 수정을 요청할 때 사용합니다.

**프로세스**:
1. 항목 선택
2. **"보완요청"** 버튼 클릭
3. 보완 사유 입력 (구체적으로)
4. 확인

**효과**:
- 해당 항목이 **보완필요** 상태로 변경
- 코치에게 **이메일 알림** 발송
- 코치가 수정 후 재제출하면 다시 검토 대기 목록에 표시

## 4. 검토사항

검토 시 다음 사항들을 **반드시** 확인해야 합니다.

### 4.1 형식적 검토사항

증빙서류의 **기본적인 형식 요건**을 확인합니다.

| 검토 항목 | 확인 내용 |
|-----------|-----------|
| **이름 일치** | 증빙서류에 기재된 이름이 응모자 이름과 일치하는지 |
| **날짜 확인** | 발급일, 취득일 등이 명확하게 기재되어 있는지 |
| **날인/직인** | 발급기관의 공식 날인 또는 직인이 있는지 |
| **문서 완결성** | 문서가 잘려있거나 훼손되지 않고 완전한 형태인지 |
| **가독성** | 스캔/촬영 품질이 내용 확인이 가능한 수준인지 |

**형식적 검토 체크리스트**:
- [ ] 이름이 응모자 정보와 일치
- [ ] 날짜(발급일/취득일)가 명확히 기재
- [ ] 발급기관의 공식 날인/직인 존재
- [ ] 문서 내용이 완전히 보임
- [ ] 위조/변조 의심 없음

### 4.2 실질적 검토사항

증빙서류가 **해당 항목의 목적에 부합**하는지 확인합니다.

| 항목 유형 | 확인 내용 |
|-----------|-----------|
| **코칭 자격증** | KCA 공인 자격증(KSC/KPC/KAC)인지, 등급이 정확한지 |
| **상담/심리치료 자격** | 자격증이 실제로 상담 또는 심리치료 관련 분야인지 |
| **학력 증빙** | 전공이 코칭/상담/심리 관련 분야인지 (해당 카테고리일 경우) |
| **코칭 경력** | 경력증명서에 코칭 업무 내용이 명시되어 있는지 |
| **코칭 시간** | 코칭 시간이 구체적으로 기재되어 있는지 |

**실질적 검토 예시**:

**예시 1: 상담/심리치료 자격 항목**
- ✓ 적합: 임상심리전문가, 상담심리사, 정신건강임상심리사
- ✗ 부적합: 요리사 자격증, 운전면허증, 일반 사무 자격증

**예시 2: 코칭/상담/심리 관련 학력**
- ✓ 적합: 심리학과, 상담학과, 교육학과(상담 전공)
- ✗ 부적합: 컴퓨터공학과, 경영학과 (해당 카테고리 기준)

**예시 3: 코칭 경력 증명**
- ✓ 적합: "OO기업 임원코칭 담당, 총 150시간 코칭 수행"
- ✗ 부적합: "영업부 사원" (코칭 업무 내용 없음)

### 4.3 검토 시 주의사항

1. **의심스러운 경우**: 컨펌하지 말고 **보완요청**으로 추가 서류 요청
2. **판단 어려운 경우**: 다른 검토자 또는 관리자와 **협의 후 결정**
3. **개인정보 보호**: 검토 과정에서 취득한 개인정보는 **외부 유출 금지**

## 5. 자주 묻는 질문 (FAQ)

### Q1: 다른 검토자가 이미 컨펌한 항목도 제가 컨펌해야 하나요?
**A**: 네, 시스템에서 설정된 검토자 수(기본 2명)가 모두 컨펌해야 검토가 완료됩니다. 각 검토자는 독립적으로 판단하여 컨펌합니다.

### Q2: 컨펌 후 실수를 발견하면 어떻게 하나요?
**A**: "리셋" 기능을 사용하여 컨펌을 초기화할 수 있습니다. 리셋하면 모든 검토자의 컨펌이 취소되고, 검토 대기 상태로 돌아갑니다.

### Q3: 코치가 증빙서류를 수정했는데 컨펌이 유지되나요?
**A**: 코치가 증빙서류를 교체하면 기존 컨펌이 자동으로 무효화되어 다시 검토해야 합니다.

### Q4: 보완요청 시 코치에게 알림이 가나요?
**A**: 네, 보완요청 시 코치에게 이메일 알림이 자동 발송됩니다. 보완 사유가 구체적으로 전달되므로, 명확하게 작성해 주세요.

### Q5: 검토 완료된 역량은 어떻게 되나요?
**A**: 검토 완료된 역량은 코치의 "전자지갑"에 저장됩니다. 코치가 다른 프로젝트에 지원할 때 해당 역량을 재검토 없이 재사용할 수 있습니다.
