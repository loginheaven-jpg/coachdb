**배점 비율**: 정량 70% / 정성 30%

**기능 버튼**:
- 가중치 설정

- 최종점수 집계
- 선발 추천
- 새로고침

**응모자 목록 컬럼**:
- 순위, 응모자(이름/이메일), 역할, 정량점수, 정성점수, 최종점수, 선발결과, 제출일, 작업

- 정성점수는 모두 "미평가" 상태
- 작업 컬럼에 여러 버튼이 있음 (상세보기, 심사, 기타)


### 3.3 선발 기능 테스트

선발 버튼(체크 아이콘)을 클릭하니 "선발 결과가 업데이트되었습니다" 메시지가 표시되고, 김철수의 선발결과가 "대기" → "선발"로 변경되었다. 선발 완료가 0/5 → 1/5로 업데이트되었다.

**발견사항**:
- 선발 버튼 클릭만으로 즉시 선발 처리됨 (확인 팝업 없음)
- 정성평가 없이도 선발 가능 → 프로세스 순서 문제 가능성

**작업 버튼 구성** (각 응모자별):
1. 상세보기 버튼 (문서 아이콘)
2. 심사 버튼 (체크 아이콘) - 선발/취소 토글
3. 삭제 버튼 (X 아이콘)


### 3.4 정성평가 입력 화면 분석

정성평가 팝업이 표시되며 다음 항목들을 입력할 수 있다:

**평가 항목 (슬라이더)**:
- 지원동기 (0-10점)
- 전문성 (0-10점)
- 역할적합성 (0-10점)
- 합계: 0점 / 30점

**추천 의견 (라디오 버튼)**:
- 강력 추천
- 추천
- 보류
- 비추천

**종합 의견**:
- 텍스트 입력 (0/1000자)
- 플레이스홀더: "지원자에 대한 종합 의견을 입력해주세요..."

**버튼**:
- 닫기
- 등록

**발견사항**:
- 정성평가 점수는 30점 만점 (지원동기 10 + 전문성 10 + 역할적합성 10)
- 배점 비율이 정량 70% / 정성 30%이므로 정성평가 30점이 최종점수의 30%를 차지
- 슬라이더 UI로 점수 입력 (직관적)


### 3.5 가중치 설정 기능 분석

가중치 설정 팝업에서 정량/정성 비율을 조정할 수 있다. 현재 설정은 정량 70%, 정성 30%이다. 두 개의 숫자 입력 필드와 취소/확인 버튼으로 구성되어 있다.

**발견사항**: 가중치 합계가 100%가 되어야 하는지 검증 로직이 있는지 확인 필요하다.


### 3.6 심사 및 선발 기능 버튼 분석

심사 및 선발 화면에서 다음 기능 버튼들이 확인되었다:

| 버튼 | 기능 | 테스트 결과 |
|------|------|------------|
| 가중치 설정 | 정량/정성 배점 비율 조정 | 정상 (70/30 기본값) |
| 정량점수 계산 | 정량평가 점수 재계산 | 미테스트 |
| 최종점수 집계 | 정량+정성 최종점수 계산 | 미테스트 |
| 선발 추천 | 자동 선발 추천 | 클릭 시 페이지 오류 발생 |
| 새로고침 | 목록 새로고침 | 미테스트 |

**발견된 문제점**: 선발 추천 버튼 클릭 시 페이지가 빈 화면으로 변경되는 오류가 발생했다.


### 3.7 과제심사 메뉴 화면 분석

과제심사 메뉴에서 "내가 심사위원으로 지정된 과제 목록"이 표시된다. 현재 4개의 과제가 표시되며, 각 과제별로 상태, 모집기간, 응모자 수, 심사하기 버튼이 있다.

**발견된 문제점**: 심사중 상태가 아닌 과제(모집중, 초안)에도 "심사하기" 버튼이 활성화되어 있다. 이는 프로세스 순서에 맞지 않으며, 심사중 상태의 과제만 심사하기 버튼이 활성화되어야 한다.


### 3.8 정성평가 입력 팝업 분석

정성평가 입력 팝업에서 다음 항목들이 확인되었다:

**평가 항목 (슬라이더 방식, 0-10점)**:
1. 지원동기 (0-10점)
2. 전문성 (0-10점)
3. 역할적합성 (0-10점)

**합계**: 0점 / 30점 (실시간 합산 표시)

**추천 의견** (라디오 버튼):
- 강력 추천
- 추천
- 보류
- 비추천

**종합 의견**: 텍스트 입력 (0/1000자)

**버튼**: 닫기, 등록

**발견된 특이사항**:
- 정성평가 항목(지원동기, 전문성, 역할적합성)이 고정되어 있음
- 과제관리자가 정성평가 항목을 커스터마이징할 수 없는 것으로 보임
- 슬라이더 UI로 점수 입력이 직관적임


### 3.9 정성평가 등록 테스트 결과

정성평가 등록이 성공적으로 완료되었다.

**등록 결과**:
- 이영희 응모자의 정성점수: 15.0 (1명 평가)
- 평가 완료 현황: 1/10 (10%)

**확인된 기능**:
- 정성평가 점수가 실시간으로 반영됨
- 평가 완료 현황이 자동 업데이트됨
- "평가가 등록되었습니다" 알림 메시지 표시


### 4.1 최종점수 집계 기능 테스트

최종점수 집계 버튼 클릭 결과, 다음과 같은 메시지가 표시되었다.

"1건의 최종 점수가 계산되었습니다. 9건은 정성평가가 없어 정량점수만 반영됩니다."

이영희 응모자의 최종점수가 50.0으로 계산되었다. 이는 정량점수 65.0의 70%와 정성점수 15.0의 30%를 합산한 결과이다 (65.0 × 0.7 + 15.0 × 0.3 = 45.5 + 4.5 = 50.0). 순위도 1위로 표시되었다.

정성평가가 없는 응모자의 경우 최종점수가 "-"로 표시되어 정량점수만으로는 최종점수가 계산되지 않는 것으로 확인되었다. 이는 모든 응모자에 대해 정성평가가 완료되어야 최종 선발이 가능함을 의미한다.


### 4.2 선발 기능 테스트 결과

이영희 응모자의 선발 버튼 클릭 결과, "선발 결과가 업데이트되었습니다" 메시지가 표시되고 선발결과가 "대기"에서 "선발"로 변경되었다. 선발 완료 현황도 1/5에서 2/5로 업데이트되었다.

현재 선발 완료 현황이 2/5로 표시되는 것은 김철수가 이미 선발 상태였기 때문이다. 작업 열에 있는 버튼들의 기능은 다음과 같이 추정된다.

| 버튼 | 기능 |
|------|------|
| 첫 번째 (반원 아이콘) | 상세보기/정성평가 입력 |
| 두 번째 (문서 아이콘) | 정성평가 입력 |
| 세 번째 (체크 아이콘) | 선발 |
| 네 번째 (X 아이콘) | 탈락 |


### 4.3 탈락 기능 테스트 결과

박민수 응모자의 탈락 버튼(X 아이콘) 클릭 결과, 선발결과가 "대기"에서 "탈락"으로 변경되었다. 탈락된 응모자의 행은 빨간색 테두리로 표시되어 시각적으로 구분된다.

현재까지 테스트된 선발 프로세스 기능은 다음과 같다.

| 기능 | 상태 | 비고 |
|------|------|------|
| 정성평가 입력 | 정상 | 슬라이더 방식, 추천 의견, 종합 의견 |
| 최종점수 집계 | 정상 | 정량 70% + 정성 30% 자동 계산 |
| 선발 | 정상 | 체크 아이콘 클릭으로 선발 처리 |
| 탈락 | 정상 | X 아이콘 클릭으로 탈락 처리 |


## 5. 응모자 화면에서 통지 확인 테스트

### 5.1 과제참여 화면 분석

과제참여 화면은 두 개의 섹션으로 구분되어 있다.

첫 번째 섹션인 "참여 과제"는 선정되어 참여중인 과제를 표시하며, 현재 "아직 없음"으로 표시되어 있다. 이는 현재 로그인한 사용자(최철영)가 아직 선발된 과제가 없음을 의미한다.

두 번째 섹션인 "미참여 과제"는 현재 모집중인 과제 목록을 표시하며, 두 개의 과제가 확인되었다. "2026년 상반기 코칭역량 종합평가 테스트 과제"는 지원하기 버튼이 활성화되어 있고, "리더십 역량 강화 그룹 코칭 프로젝트"는 "지원완료" 상태로 보기/수정 버튼이 표시되어 있다.

선발 결과 통지와 관련하여 다음 사항이 확인되었다. 선발된 응모자의 경우 "참여 과제" 섹션에 해당 과제가 표시되어야 하나, 현재 테스트 사용자는 선발된 과제가 없어 확인이 불가능하다. 또한 최신소식이나 알림 기능이 화면에서 직접 확인되지 않으며, 메일 통지 기능은 별도 테스트가 필요하다.


### 5.2 대시보드 화면 분석

대시보드 화면에는 4개의 요약 카드가 표시되어 있다.

| 카드 | 내용 |
|------|------|
| 내 지원서 | 1건 |
| 관리 중인 과제 | 4개 |
| 증빙 검토 | 0건 대기 |
| 심사 대상 과제 | 4개 |

빠른 시작 섹션에는 "모집중인 과제 보기", "내 역량 정보 관리", "새 과제 만들기" 버튼이 있다.

**중요 발견: 최신소식/알림 기능 부재**

대시보드에서 다음 기능이 확인되지 않았다.
- 최신소식 섹션
- 알림 아이콘/뱃지
- 보완요청 알림
- 선발/탈락 결과 통지

사용자 요구사항에 따르면 응모자는 "최근소식 화면과 응모한 과제내역, 그리고 메일로 보완요청이 온 것을 쉽게 알 수 있어야 한다"고 명시되어 있으나, 현재 대시보드에는 이러한 알림 기능이 구현되어 있지 않다.

### 6.4 개선 제안

**우선순위 높음 (기능 추가)**

1. **최신소식/알림 기능 추가**
   - 대시보드에 "최신소식" 섹션 추가
   - 알림 아이콘에 읽지 않은 알림 수 뱃지 표시
   - 알림 유형: 보완요청, 선발/탈락 결과, 검토 완료 등

2. **메일 통지 기능**
   - 보완요청 시 응모자에게 메일 발송
   - 선발/탈락 결정 시 응모자에게 메일 발송
   - 검토 완료 시 과제관리자에게 메일 발송

**우선순위 중간 (UX 개선)**

3. **심사 화면 개선**
   - 작업 열의 버튼에 툴팁 추가 (현재 아이콘만으로 기능 파악 어려움)
   - 선발/탈락 버튼 클릭 전 확인 팝업 추가

4. **검토 인원 설정 기능**
   - 과제 생성/수정 시 필수 검토 인원 설정 필드 추가
   - 검토 완료 조건 명확화 (N명 컨펌 시 확정)

5. **과제 상태 흐름 시각화**
   - 현재 상태와 다음 단계를 명확히 표시
   - 상태 변경 조건 안내 (예: "모든 증빙 검토 완료 시 검토종료 가능")

**우선순위 낮음 (편의 기능)**

6. **일괄 처리 기능**
   - 다수 응모자 일괄 선발/탈락
   - 다수 증빙 일괄 컨펌

7. **통계 대시보드**
   - 과제별 검토 진행률
   - 심사 완료율
   - 선발/탈락 현황
