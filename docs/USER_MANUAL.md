## 1. 시스템 개요

### 1.1 과제관리자 권한
과제관리자(PROJECT_MANAGER)는 다음 작업을 수행할 수 있습니다:
- 과제 생성 및 수정
- 설문 항목 및 배점 설정
- 지원서 조회 및 관리
- 정량/정성 평가 진행
- 최종 선발 결정
- 검토자 배정

### 1.2 과제 상태 흐름
```
초안(DRAFT) → 승인완료(READY) → 모집중 → 심사중 → 완료/종료
```

| 상태 | 설명 |
|------|------|
| **초안** | 과제 설정 중. 외부 비공개 |
| **승인완료** | 배점 검증 완료. 모집 시작일에 자동 공개 |
| **모집중** | 지원서 접수 중 |
| **심사중** | 모집 마감 후 평가 진행 |
| **완료** | 선발 확정 완료 |

## 2. 과제 개설

### 2.1 새 과제 만들기
1. 상단 메뉴에서 **[과제 관리]** 클릭
2. **[새 과제 만들기]** 버튼 클릭
3. 기본 정보 입력:

| 필드 | 필수 | 설명 |
|------|------|------|
| 과제명 | O | 과제 제목 (예: "2024년 코치 인증 심사") |
| 지원 프로그램명 | X | 상위 프로그램명 |
| 모집 기간 | O | 지원서 접수 시작일 ~ 마감일 |
| 과제 수행 기간 | X | 실제 사업 수행 기간 |
| 최대 인원 | O | 선발 가능 최대 인원 |
| 담당자 | X | 과제 담당 관리자 |

4. **[저장]** 클릭 → 초안 상태로 저장

### 2.2 과제 수정
1. **[과제 관리]** → 해당 과제의 **[수정]** 클릭
2. 좌측 탭에서 수정할 영역 선택:
   - **기본정보**: 과제명, 기간 등
   - **역량항목**: 설문 항목 및 배점 설정

---

## 3. 설문 구성 및 배점 설정

### 3.1 역량항목 추가
1. 과제 수정 화면에서 **[역량항목]** 탭 선택
2. **[항목 추가]** 버튼 클릭
3. 항목 정보 입력:

| 필드 | 설명 |
|------|------|
| 항목 코드 | 고유 식별 코드 (예: CERT-001) |
| 항목명 | 항목 이름 (예: "코칭 자격증") |
| 배점 | 해당 항목의 최대 점수 |
| 입력 유형 | 텍스트, 숫자, 선택형 등 |
| 반복 가능 여부 | 여러 개 입력 허용 시 체크 |

### 3.2 등급별 배점 설정
각 항목에 대해 세부 점수 기준을 설정할 수 있습니다.

1. 항목 우측의 **[배점 설정]** 버튼 클릭
2. 등급 유형 선택:

| 등급 유형 | 용도 | 예시 |
|----------|------|------|
| **문자열** | 특정 값에 따른 점수 | KSC=10점, KPC=5점 |
| **숫자 범위** | 숫자 구간별 점수 | 1000시간 이상=10점 |
| **복수선택** | 선택 항목 기반 점수 | 코칭 포함 시 5점 |
| **파일 유무** | 파일 제출 여부 | 있음=10점, 없음=0점 |

3. **문자열 등급** 설정 시:
   - 매칭 방식 선택: **정확히 일치** 또는 **포함**
   - 예: "KSC" 포함 선택 시 "KSC-2024-001"도 매칭

4. **숫자 범위 등급** 설정 시:
   - 범위별 점수 지정 (예: 100 이상=10점, 50~99=7점)
   - **증빙 감점**: 증빙 파일 미제출 시 감점 설정 가능

5. **[적용]** 클릭하여 저장

### 3.3 점수 계산 기준
점수를 계산할 때 어떤 값을 기준으로 할지 선택합니다:

| 기준 | 설명 |
|------|------|
| **지원자 입력값** | 지원자가 직접 입력한 값 |
| **회원정보** | 회원 프로필에서 가져온 값 (인증번호, 소속, 코칭분야) |

### 3.4 저장 방식

| 저장 유형 | 배점 검증 | 상태 변경 |
|----------|----------|----------|
| **임시저장** | 검증 안함 | 초안 유지 |
| **정식저장** | 총점 100점 검증 | 승인완료로 전환 |

> **주의**: 정식저장하려면 모든 항목의 배점 합계가 정확히 **100점**이어야 합니다.

## 4. 지원서 접수 관리

### 4.1 지원서 목록 조회
1. **[과제 관리]** → 해당 과제 클릭
2. **[지원서 목록]** 탭에서 접수된 지원서 확인

### 4.2 지원서 상태

| 상태 | 설명 |
|------|------|
| **제출됨** | 지원자가 제출 완료 |
| **보완요청** | 서류 보완 요청됨 |
| **보완완료** | 지원자가 보완 제출 |
| **심사완료** | 검토 완료 |

### 4.3 지원서 상세 보기
- 지원서 클릭하여 상세 내용 확인
- 제출된 역량 항목별 값과 증빙 파일 조회
- 검증 상태 확인 (승인/반려/대기)

### 4.4 접수 마감
모집 기간이 종료되면 자동으로 접수가 마감됩니다.
필요 시 수동으로 **[접수 마감]** 처리할 수 있습니다.

## 5. 심사 진행

### 5.1 심사 개요
심사는 **정량평가**와 **정성평가** 두 단계로 진행됩니다.

```
정량평가(자동) + 정성평가(검토자) = 최종점수
```

### 5.2 정량평가 (자동 점수 계산)
1. **[심사 관리]** 메뉴 진입
2. **[정량점수 계산]** 버튼 클릭
3. 시스템이 자동으로 배점 기준에 따라 점수 산출

> 정량점수는 설문 항목별 배점 설정에 따라 자동 계산됩니다.

### 5.3 정성평가 (검토자 평가)
1. 검토자가 각 지원서에 대해 평가 점수 입력
2. 평가 항목:
   - **점수**: 0~100점
   - **의견**: 평가 소견
   - **추천 여부**: 선발 추천 의견

### 5.4 배점 비율 설정
정량/정성 평가의 반영 비율을 조정할 수 있습니다.

| 항목 | 기본값 | 설명 |
|------|--------|------|
| 정량평가 비율 | 70% | 자동 계산 점수 반영 비율 |
| 정성평가 비율 | 30% | 검토자 평가 반영 비율 |

**설정 방법**:
1. **[심사 관리]** → **[배점 비율 설정]**
2. 비율 조정 후 **[저장]**

### 5.5 최종점수 집계
1. **[최종점수 집계]** 버튼 클릭
2. 계산 공식:
   ```
   최종점수 = (정량점수 × 정량비율) + (정성평균 × 정성비율)
   ```
3. 점수순 자동 순위 부여

### 5.6 심사 현황 대시보드
심사 진행 상황을 한눈에 확인할 수 있습니다:
- 총 지원자 수
- 평가 완료율
- 선발 인원 현황
- 평균 점수

---

## 6. 최종 선발

### 6.1 선발 추천
1. **[심사 관리]** → **[선발 추천]** 클릭
2. 시스템이 최대 인원 기준으로 상위 점수자 자동 추천
3. 추천 결과 확인:
   - 선발 추천: 상위 N명 (N = 최대 인원)
   - 탈락 추천: 나머지

### 6.2 수동 선발/탈락 처리
자동 추천과 별개로 개별 조정 가능합니다.

1. 지원자 목록에서 해당 지원자 선택
2. **[선발]** 또는 **[탈락]** 버튼 클릭
3. 선발 결과 변경 저장

| 선발 상태 | 표시 | 설명 |
|----------|------|------|
| **대기** | 회색 | 아직 결정 안됨 |
| **선발** | 녹색 | 최종 선발 확정 |
| **탈락** | 적색 | 탈락 처리 |

### 6.3 선발 확정
1. 모든 지원자의 선발/탈락 결정 완료 확인
2. **[선발 확정]** 버튼 클릭
3. 확인 팝업에서 **[확정]** 클릭

> **주의**: 선발 확정 시 지원자에게 결과 알림이 자동 발송됩니다.

### 6.4 선발 결과 조회
확정 후 선발 결과를 조회할 수 있습니다:
- 선발자 명단
- 탈락자 명단
- 점수 및 순위 정보

## 7. 검토자 배정

### 7.1 검토자 추가
1. **[과제 상세]** → **[검토자 관리]** 탭
2. **[검토자 추가]** 버튼 클릭
3. 사용자 검색 후 선택
4. **[추가]** 클릭

> 검토자로 배정된 사용자는 해당 과제의 지원서를 평가할 수 있습니다.

### 7.2 검토자 제거
1. 검토자 목록에서 해당 검토자 선택
2. **[제거]** 버튼 클릭

### 7.3 검토자 권한
배정된 검토자는 다음 작업을 수행할 수 있습니다:
- 지원서 조회
- 정성평가 점수 입력
- 평가 의견 작성

> 검토자는 선발 결정 권한이 **없습니다**. 선발은 과제관리자만 가능합니다.

## 1. 검토자 역할 개요

### 1.1 검토자(REVIEWER)란?
검토자는 과제관리자에 의해 특정 과제에 배정되어 지원서를 평가하는 역할입니다.

### 1.2 검토자 권한 범위

| 가능한 작업 | 불가능한 작업 |
|------------|--------------|
| 배정된 과제의 지원서 조회 | 과제 생성/수정 |
| 정성평가 점수 입력 | 선발/탈락 결정 |
| 평가 의견 작성 | 다른 검토자 배정 |
| 자신의 평가 수정 | 배점 기준 변경 |

### 1.3 평가 프로세스 흐름
```
과제 배정 → 지원서 조회 → 평가 입력 → 제출 → (수정 가능) → 최종 집계
```

## 2. 배정된 과제 확인

### 2.1 대시보드에서 확인
1. 로그인 후 **대시보드** 접속
2. **[내 평가 과제]** 섹션에서 배정된 과제 목록 확인
3. 과제별 평가 진행률 표시

### 2.2 과제 목록에서 확인
1. **[과제 목록]** 메뉴 클릭
2. **[내 평가 과제]** 필터 선택
3. 배정된 과제만 필터링되어 표시

### 2.3 과제 상세 정보
과제 클릭 시 확인 가능한 정보:
- 과제명 및 기간
- 평가 대상 지원서 수
- 나의 평가 완료율
- 평가 마감일

---

## 3. 지원서 평가하기

### 3.1 지원서 목록 조회
1. 배정된 과제 클릭
2. **[지원서 목록]** 에서 평가 대상 지원서 확인
3. 평가 상태 확인:
   - **미평가**: 아직 평가하지 않음
   - **평가완료**: 평가 제출 완료

### 3.2 지원서 상세 보기
지원서를 클릭하면 다음 정보를 확인할 수 있습니다:

| 영역 | 내용 |
|------|------|
| **기본정보** | 지원자 성명, 연락처 등 |
| **역량항목** | 제출된 역량 정보 및 증빙 |
| **정량점수** | 자동 계산된 점수 (참고용) |
| **검증상태** | 각 항목의 검증 승인 여부 |

### 3.3 증빙 파일 확인
- 각 역량항목에 첨부된 파일 확인
- 파일 미리보기 또는 다운로드
- 증빙 적절성 판단

### 3.4 평가 점수 입력
1. 지원서 상세 화면에서 **[평가하기]** 버튼 클릭
2. 평가 항목 입력:

| 항목 | 입력 범위 | 설명 |
|------|----------|------|
| **평가 점수** | 0~100점 | 지원자에 대한 종합 평가 점수 |
| **평가 의견** | 텍스트 | 평가 근거 및 소견 |
| **선발 추천** | 추천/비추천/보류 | 선발 여부에 대한 의견 |

3. **[제출]** 버튼 클릭

### 3.5 평가 기준 참고사항
공정한 평가를 위해 다음 사항을 고려하세요:

- **객관성**: 제출된 자료 기반으로 평가
- **일관성**: 모든 지원자에게 동일 기준 적용
- **근거 명시**: 평가 의견에 구체적 근거 기술

## 4. 평가 수정 및 완료

### 4.1 평가 수정
최종 집계 전까지 평가를 수정할 수 있습니다.

1. 해당 지원서 상세 화면 진입
2. **[평가 수정]** 버튼 클릭
3. 점수 또는 의견 수정
4. **[저장]** 클릭

### 4.2 평가 현황 확인
**[내 평가 현황]** 에서 진행 상황을 확인할 수 있습니다:
- 총 평가 대상: N건
- 평가 완료: N건
- 미평가: N건
- 완료율: N%

### 4.3 평가 완료 체크리스트
모든 평가를 완료했는지 확인하세요:

- [ ] 모든 지원서에 점수 입력 완료
- [ ] 각 평가에 의견 작성 완료
- [ ] 선발 추천 의견 입력 완료

### 4.4 평가 마감
- 과제관리자가 최종점수 집계를 실행하면 평가가 마감됩니다
- 마감 후에는 평가 수정이 불가능합니다
- 마감 전 반드시 모든 평가를 완료해주세요

---

## 부록: 자주 묻는 질문 (FAQ)

### Q1. 배정된 과제가 보이지 않습니다.
**A**: 과제관리자가 아직 검토자로 배정하지 않았을 수 있습니다. 담당자에게 문의하세요.

### Q2. 평가를 제출했는데 수정이 안 됩니다.
**A**: 최종점수 집계가 완료되면 수정이 불가합니다. 집계 전이라면 관리자에게 문의하세요.

### Q3. 다른 검토자의 평가를 볼 수 있나요?
**A**: 아니요, 각 검토자의 평가는 독립적으로 진행되며 다른 검토자의 평가는 조회할 수 없습니다.

### Q4. 점수 기준이 궁금합니다.
**A**: 과제별로 배점 기준이 다를 수 있습니다. 과제 상세 정보에서 평가 기준을 확인하거나 과제관리자에게 문의하세요.

### Q5. 증빙 파일이 열리지 않습니다.
**A**: 파일 형식이 지원되지 않거나 파일이 손상되었을 수 있습니다. 다운로드 후 확인하거나 과제관리자에게 문의하세요.
