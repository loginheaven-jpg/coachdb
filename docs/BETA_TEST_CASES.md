# PPMS 베타테스트 케이스

## 테스트 환경
- URL: https://ppms.netlify.app
- 브라우저: Chrome (권장), Firefox, Safari
- 테스트 계정: 개별 제공

---

## 1. 인증 기능

### TC-AUTH-01: 회원가입
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | /register 페이지 접속 | 회원가입 폼 표시 |
| 2 | 이름, 이메일, 비밀번호, 전화번호 입력 | 입력값 반영 |
| 3 | 회원가입 버튼 클릭 | 성공 메시지 후 로그인 페이지로 이동 |

### TC-AUTH-02: 로그인
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | /login 페이지 접속 | 로그인 폼 표시 |
| 2 | 이메일, 비밀번호 입력 | 입력값 반영 |
| 3 | 로그인 버튼 클릭 | 대시보드로 이동 |

### TC-AUTH-03: 로그아웃
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 로그인 상태에서 상단 우측 아바타 클릭 | 드롭다운 메뉴 표시 |
| 2 | 로그아웃 클릭 | 로그인 페이지로 이동 |

### TC-AUTH-04: 비밀번호 재설정
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | /forgot-password 페이지 접속 | 비밀번호 재설정 폼 표시 |
| 2 | 등록된 이메일 입력 후 제출 | 이메일 발송 안내 메시지 |

## 2. 과제참여

### TC-PART-01: 과제참여 목록 조회
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 상단 메뉴에서 "과제참여" 클릭 | /projects 페이지로 이동 |
| 2 | 페이지 로드 완료 | 모집중인 과제 목록 표시 |
| 3 | 통계 카드 확인 | 모집중 과제, 지원완료, 미지원 수 표시 |

### TC-PART-02: 과제 지원하기
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 모집중인 과제의 "지원하기" 버튼 클릭 | 지원서 작성 페이지로 이동 |
| 2 | 기본정보 탭에서 정보 확인/수정 | 프로필에서 가져온 정보 표시 |
| 3 | 설문항목 탭 클릭 | 설문 항목 목록 표시 |
| 4 | 각 설문항목 작성 | 입력값 저장됨 |
| 5 | 미리보기 클릭 | 작성 내용 미리보기 표시 |
| 6 | 제출 버튼 클릭 | "지원완료" 메시지, 과제목록으로 이동 |

### TC-PART-03: 본인 과제에 지원
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 과제관리자로 로그인 | 대시보드 표시 |
| 2 | "과제참여" 메뉴에서 본인이 만든 과제 확인 | 본인 과제도 목록에 표시됨 |
| 3 | 본인 과제의 "지원하기" 버튼 확인 | 지원하기 버튼 활성화됨 |
| 4 | 지원하기 버튼 클릭 | 지원서 작성 페이지로 이동 가능 |

### TC-PART-04: 지원서 수정
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 과제참여 목록에서 "지원완료" 태그 클릭 | 지원서 보기 페이지 |
| 2 | (모집기간 내) "수정" 버튼 클릭 | 수정 모드로 전환 |
| 3 | 내용 수정 후 저장 | 수정 완료 메시지 |

## 3. 과제관리

### TC-MGMT-01: 과제관리 메뉴 접근
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 과제관리자 계정으로 로그인 | "과제관리" 메뉴 표시됨 |
| 2 | "과제관리" 클릭 | /projects/manage 페이지 이동 |
| 3 | 일반 코치 계정으로 로그인 | "과제관리" 메뉴 없음 |

### TC-MGMT-02: 새 과제 생성
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | "새 과제 생성" 버튼 클릭 | 과제 생성 폼 표시 |
| 2 | 과제명, 설명, 정원, 기간 입력 | 입력값 반영 |
| 3 | 설문항목 추가 | 항목 추가됨 |
| 4 | 저장 버튼 클릭 | 과제 생성 완료, 목록으로 이동 |

### TC-MGMT-03: 과제 수정
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 과제 목록에서 "수정" 버튼 클릭 | 수정 폼 표시 |
| 2 | 내용 수정 | 변경사항 반영 |
| 3 | 저장 클릭 | 수정 완료 메시지 |

### TC-MGMT-04: 응모자 조회
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 과제 상세에서 "응모자" 버튼 클릭 | 응모자 목록 표시 |
| 2 | 응모자 클릭 | 지원서 상세 내용 표시 |
| 3 | 선발/탈락 처리 | 상태 변경됨 |

### TC-MGMT-05: 수퍼어드민 전체 과제 조회
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 수퍼어드민으로 로그인 | 대시보드 표시 |
| 2 | "과제관리" 메뉴 클릭 | 모든 사용자의 과제 목록 표시 |
| 3 | 다른 사용자 과제 수정 가능 | 수정 권한 있음 |

## 4. 세부정보(역량) 관리

### TC-COMP-01: 세부정보 페이지 접근
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 상단 아바타 클릭 | 드롭다운 메뉴 표시 |
| 2 | "세부정보 관리" 클릭 | /coach/competencies 페이지 이동 |

### TC-COMP-02: 자격증 추가
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | "자격증" 섹션 펼치기 | 자격증 항목 목록 표시 |
| 2 | "+ 추가" 버튼 클릭 | 추가 모달 표시 |
| 3 | 자격증 정보 입력 | 입력값 반영 |
| 4 | 증빙서류 업로드 | 파일 첨부됨 |
| 5 | 추가 버튼 클릭 | 자격증 추가 완료 |

### TC-COMP-03: 파일 Drag & Drop
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 세부정보 관리 페이지 접속 | 페이지 표시 |
| 2 | 파일을 페이지에 드래그 | **풀스크린으로 열리지 않음** |
| 3 | Upload 영역에 드롭 | 파일 첨부됨 |

### TC-COMP-04: 복수 항목 추가 (자격증 등)
| 단계 | 동작 | 예상 결과 |
|------|------|-----------|
| 1 | 자격증 항목 추가 (1개) | 1번째 자격증 표시 |
| 2 | "+ 추가" 버튼 클릭 (2개째) | 2번째 자격증 추가됨 |
| 3 | 개수 제한 확인 | 최대 개수 도달 시 버튼 비활성화 |

## 6. 데이터 연동

### TC-SYNC-01: 세부정보 → 설문항목 연동

| 단계 | 동작 | 예상 결과 |
|------|------|-----------|

| 2 | 저장 클릭 | "세부정보에도 반영됩니다" 확인창 |
| 3 | 확인 클릭 | 세부정보 페이지에서도 변경됨 |
