# 1. 사용자 인증 및 권한 관리

## TC-1.1: 회원가입 및 로그인

### TC-1.1.1: 일반 코치 회원가입
**사전조건**: 신규 사용자
**테스트 단계**:
1. `/register` 페이지 접속
2. 필수 정보 입력:
   - 이메일
   - 비밀번호 (최소 8자, 영문+숫자 포함)
   - 이름 (최소 2자)
   - 역할: COACH가 기본 선택됨 (항상 포함)
   - 전화번호
   - 생년 (예: 1985)
   - 성별 선택
   - 코치 자격증 번호 (필수)
   - 주소 (시/군/구 단위)
   - 코칭 분야 (복수 선택, 최소 1개)
3. 선택 정보 입력:
   - 대면코칭 가능 지역
4. 회원가입 버튼 클릭
**예상 결과**:
- 회원가입 성공 메시지 표시
- 자동 로그인 후 `/coach/dashboard`로 이동
- COACH 역할이 즉시 부여됨
- 비밀번호 형식 오류 시 "영문과 숫자를 포함해야 합니다" 메시지

### TC-1.1.2: 관리자 권한 회원가입 (승인 필요)
**사전조건**: 신규 사용자
**테스트 단계**:
1. `/register` 페이지 접속
2. 역할 선택: "프로젝트 관리자(PROJECT_MANAGER)" 선택
3. 회원가입 완료
4. **별도 창**: SUPER_ADMIN 계정으로 로그인
5. `/admin/users` → 권한 요청 목록 확인
6. 해당 사용자의 권한 요청 승인
7. **원래 창**: 로그아웃 후 다시 로그인
**예상 결과**:
- 초기에는 COACH 역할만 부여됨
- 승인 전: 관리자 메뉴 접근 불가
- 승인 후 로그인: PROJECT_MANAGER 역할 포함 확인
- `/admin/dashboard` 접근 가능

### TC-1.1.3: 중복 이메일 회원가입 방지
**사전조건**: 이미 등록된 이메일 존재
**테스트 단계**:
1. 기존 이메일로 회원가입 시도
**예상 결과**:
- "이미 등록된 이메일입니다" 오류 메시지

### TC-1.1.4: 로그인 실패 - 잘못된 비밀번호
**테스트 단계**:
1. 올바른 이메일, 잘못된 비밀번호 입력
**예상 결과**:
- "이메일 또는 비밀번호가 올바르지 않습니다" 오류

### TC-1.1.5: 비밀번호 찾기
**테스트 단계**:
1. `/forgot-password` 접속
2. 등록된 이메일 입력
3. 이메일 수신 확인
4. 링크 클릭 → `/reset-password` 이동
5. 새 비밀번호 입력
6. 새 비밀번호로 로그인
**예상 결과**:
- 비밀번호 재설정 이메일 수신
- 새 비밀번호로 로그인 성공

### TC-1.2.2: VERIFIER 역할 메뉴 접근

1. 대시보드 확인

**예상 결과**:
- ✅ `/admin/verifications` 접근 가능
- ✅ 검증 대기 목록 표시

### TC-1.2.3: PROJECT_MANAGER 역할 메뉴 접근
**사전조건**: PROJECT_MANAGER 역할 가진 사용자 로그인
**테스트 단계**:
1. 관리자 대시보드 접근
2. 과제 생성/수정 메뉴 확인
**예상 결과**:
- ✅ `/admin/projects/create` 접근 가능
- ✅ 본인이 생성한 과제 편집 가능
- ❌ `/admin/users` 접근 불가

### TC-1.2.4: SUPER_ADMIN 전체 권한 확인
**사전조건**: SUPER_ADMIN 역할 가진 사용자 로그인
**테스트 단계**:
1. 모든 관리 메뉴 접근 시도
**예상 결과**:
- ✅ 모든 페이지 접근 가능
- ✅ `/admin/competency-items` 접근 가능
- ✅ `/admin/users` 접근 가능

### TC-1.2.5: 어드민의 사용자 역할 부여
**사전조건**: SUPER_ADMIN 로그인
**테스트 단계**:
1. `/admin/users` 접속
2. 특정 사용자 선택
3. VERIFIER 역할 추가
4. 저장
5. **다른 브라우저**: 해당 사용자로 로그인
6. `/admin/verifications` 접근 시도
**예상 결과**:
- 역할 저장 성공
- 해당 사용자가 검증 메뉴에 접근 가능

# 2. 프로필 관리

## TC-2.1: 기본 프로필

### TC-2.1.1: 기본정보 수정
**사전조건**: 로그인된 사용자
**테스트 단계**:
1. `/profile/edit` 접속
2. "기본정보" 탭 선택
3. 이름, 전화번호, 주소 수정
4. 저장 버튼 클릭
5. 페이지 새로고침
**예상 결과**:
- 저장 성공 메시지
- 새로고침 후에도 수정된 정보 유지

### TC-2.1.2: 비밀번호 변경
**테스트 단계**:
1. 프로필 편집 페이지 `/profile/edit` 접속
2. 기본정보 탭 하단의 "비밀번호 변경" 섹션으로 이동
3. 새 비밀번호 입력 (최소 6자)
4. 비밀번호 확인 필드에 동일하게 입력
5. "비밀번호 변경" 버튼 클릭
6. 로그아웃 후 새 비밀번호로 로그인
**예상 결과**:
- "비밀번호가 변경되었습니다!" 성공 메시지
- 새 비밀번호로 로그인 성공
- 비밀번호 불일치 시 "비밀번호가 일치하지 않습니다!" 오류

### TC-2.1.3: 비밀번호 유효성 검사
**테스트 단계**:
1. 5자 이하의 짧은 비밀번호 입력 시도
**예상 결과**:
- "비밀번호는 최소 6자 이상이어야 합니다!" 오류

## TC-2.2: 세부정보 (역량 관리)

### TC-2.2.1: 프로필 편집에서 세부정보 탭 접근
**사전조건**: COACH 로그인
**테스트 단계**:
1. `/profile/edit` 접속
2. "세부정보" 탭 클릭
**예상 결과**:
- UnifiedCompetencyPage가 임베딩되어 표시
- 모든 카테고리 표시 (기본정보, 자격증, 학력, 코칭관련 연수, 역량이력, 기타)

### TC-2.2.2: 학력 추가 (EDUCATION 카테고리)
**테스트 단계**:
1. 세부정보 탭 → "학력" 섹션 펼치기
2. "+ 추가" 버튼 클릭
3. 학위 레벨 선택 (박사/석사/학사)
4. 전공명 입력
5. 증빙 파일 업로드 (PDF)
6. 저장
**예상 결과**:
- 학력 항목이 목록에 추가됨
- 업로드된 파일 아이콘 표시
- 검증 상태: "검토중"

### TC-2.2.3: 코칭관련 연수 추가
**테스트 단계**:
1. "코칭관련 연수" 섹션 펼치기
2. "+ 연수 추가" 버튼 클릭
3. 교육명, 교육기관, 이수일, 교육시간 입력
4. 수료증 파일 업로드
5. 저장
**예상 결과**:
- 연수 테이블에 새 행 추가됨
- 입력한 모든 정보 표시

### TC-2.2.4: 자격증 추가 (CERTIFICATION 카테고리)
**테스트 단계**:
1. "자격증" 섹션 펼치기
2. "KCA 코칭관련 자격증" 항목의 "+ 추가" 클릭
3. 자격증 명칭 입력
4. 자격증 파일 업로드
5. 저장
**예상 결과**:
- 자격증 카드가 추가됨
- 파일 다운로드 가능

### TC-2.2.5: 역량 수정
**사전조건**: 기존 역량 항목 존재
**테스트 단계**:
1. 기존 역량 항목의 편집 아이콘 클릭
2. 값 수정
3. 저장
**예상 결과**:
- 수정 성공
- 수정된 값 반영
- 검증 상태가 "검토중"으로 재설정

### TC-2.2.6: 역량 삭제
**테스트 단계**:
1. 역량 항목의 삭제 아이콘 클릭
2. 확인 다이얼로그에서 "삭제" 클릭
**예상 결과**:
- 항목이 목록에서 제거됨

### TC-2.2.7: 파일 다운로드
**사전조건**: 파일이 첨부된 역량 존재
**테스트 단계**:
1. 파일명 링크 클릭
**예상 결과**:
- 파일 다운로드 시작
- 원본 파일명으로 다운로드

# 3. 과제(프로젝트) 관리

## TC-3.1: 과제 생성

### TC-3.1.1: 신규 과제 생성
**사전조건**: PROJECT_MANAGER 또는 SUPER_ADMIN 로그인
**테스트 단계**:
1. `/admin/projects/create` 접속
2. 과제명, 지원사업명 입력
3. 모집 기간 설정
4. 과제 기간 설정
5. 최대 참여인원 입력
6. 저장
**예상 결과**:
- 과제 생성 성공
- 상태: DRAFT (초안)
- 과제 상세 페이지로 이동

### TC-3.1.2: 과제 수정 - 탭 구조 확인
**사전조건**: 생성된 과제 존재
**테스트 단계**:
1. `/admin/projects/{id}/edit` 접속
2. "기본정보" 탭 확인
3. "설문항목 (XX/100점)" 탭 확인
**예상 결과**:
- 두 개의 탭이 표시됨
- 설문항목 탭에 현재 배점 표시

---

## TC-3.2: 설문항목 구성

### TC-3.2.1: 역량 항목 추가
**테스트 단계**:
1. 과제 수정 → "설문항목" 탭 선택
2. "항목 추가" 버튼 클릭
3. 역량 항목 선택 (예: "코칭/상담/심리 관련 최종학력")
4. 배점 입력 (예: 15점)
5. 필수 여부 설정
6. 증빙 필요 수준 설정
7. 저장
**예상 결과**:
- 항목이 설문 목록에 추가됨
- 총점이 업데이트됨 (XX/100점)

### TC-3.2.2: 100점 검증
**테스트 단계**:
1. 여러 항목 추가하여 총점 100점 미만 상태로 만듦
2. 과제 확정(Finalize) 시도
**예상 결과**:
- "총 배점이 100점이어야 합니다" 오류 메시지

### TC-3.2.3: 과제 정식저장 (Finalize)
**사전조건**: 총점 100점인 설문 구성 + 과제 기간(예정) 입력
**테스트 단계**:
1. 기본정보 탭에서 "정식저장 조건" 섹션 확인:
   - ✅ 과제 기간 입력 완료
   - ✅ 설문 점수 100/100점
2. "정식저장" 버튼 클릭
**예상 결과**:
- 상태: DRAFT → READY
- "정식저장 되었습니다. 모집시작일에 자동으로 공개됩니다." 메시지
- 과제 상세 페이지로 이동

### TC-3.2.3.1: 정식저장 조건 미충족
**테스트 단계**:
1. 과제 기간을 입력하지 않거나 설문 점수가 100점 미만인 상태
2. "정식저장" 버튼 클릭 시도
**예상 결과**:
- 정식저장 조건 표시 영역에 Warning 표시
- 미충족 조건 목록 표시 (예: "설문항목 탭에서 배점을 100점으로 맞춰주세요")

### TC-3.2.4: 모집 시작
**사전조건**: READY 상태 과제
**테스트 단계**:
1. 모집 시작일이 되면 상태 확인
**예상 결과**:
- 상태: READY → RECRUITING
- 코치들에게 과제 목록에 표시됨

### TC-3.2.5: 과제 재편집 (Unpublish)
**테스트 단계**:
1. READY 상태에서 "초안으로 되돌리기" 클릭
**예상 결과**:
- 상태: READY → DRAFT
- 설문항목 수정 가능

# 4. 지원서 제출 및 관리

## TC-4.1: 지원서 작성

### TC-4.1.1: 지원서 페이지 접근
**사전조건**: COACH 로그인, 모집중인 과제 존재
**테스트 단계**:
1. `/projects` 접속
2. 모집중인 과제 클릭
3. "지원하기" 버튼 클릭
**예상 결과**:
- `/coach/projects/{id}/apply` 페이지로 이동

**예상 결과**:

**테스트 단계**:
1. VERIFIER A 로그인 → 검증 확인
2. 코치 화면에서 상태 확인

4. 코치 화면에서 상태 확인
**예상 결과**:
- 1명 확인 후: "검토중" (또는 부분 승인 표시)
- 2명 확인 후: "검증완료" (전역 승인)

### TC-6.1.6: 검증 취소
**테스트 단계**:
1. VERIFIER 로그인

**예상 결과**:
- 검증 기록 삭제
- 상태가 다시 검토중으로 변경
